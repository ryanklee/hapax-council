# Published Artifacts — DOI Index

**Owner:** epsilon (V5 weave wk1 d4 — `docs/published-artifacts/` epsilon-canonical).
**Spec:** V5 weave inflection 2026-04-25T15:08Z § 2.4 — *Cross-cutting authoring infra: DOI index*.
**Lifecycle:** an artifact lands here after the publish-bus completes
its first publish-event for that artifact; the entry tracks the
canonical DOI (Zenodo-minted in V5 weave wk2+) plus the per-surface
landing record across the 13 FULL_AUTO surfaces.

This directory is the single source of truth for "what has Hapax
published, where, and how to cite." Every entry is created and
maintained by automated tools (per the
`feedback_full_automation_or_no_engagement` directive); operator
intervention is limited to one-time-bootstrap acknowledgements
recorded in artifact frontmatter.

## Directory shape

```
docs/published-artifacts/
├── README.md                              # this file
├── _doi_index.yaml                        # auto-generated — DOI mapping
├── _surface_landing_log.jsonl             # per-publish-event landing record
└── <artifact-slug>/                       # per-artifact subdir
    ├── source.md                          # canonical Markdown source
    ├── render.pdf                         # Pandoc + Eisvogel PDF render (when applicable)
    ├── metadata.yaml                      # frontmatter + DOI + surface-targets
    └── attribution.yaml                   # rendered AttributionBlock for traceability
```

Each per-artifact subdir is created by the publisher kit on
first-publish-event. The `metadata.yaml` carries:

- `slug`: artifact identifier (matches Manifesto's frontmatter slug pattern)
- `doi`: Zenodo-minted DOI (or null pre-mint)
- `byline_variant`: BylineVariant (per `shared.attribution_block.SURFACE_DEVIATION_MATRIX`)
- `unsettled_variant`: UnsettledContributionVariant
- `non_engagement_form`: NonEngagementForm (SHORT or LONG)
- `surfaces_targeted`: list of surface ids the artifact was queued to
- `surfaces_landed`: list of surface ids where the publish-event succeeded
- `published_at`: ISO-8601 timestamp of first successful publish
- `polysemic_audit_passed`: bool (regression-pin against drift)

## Indexing schema

`_doi_index.yaml` is auto-regenerated by
`scripts/regenerate-doi-index.py` (forthcoming, V5 weave wk2). The
schema mirrors citation-file-format `Entity` author lists; downstream
consumers are CITATION.cff regen, the operator's research dashboard,
and the post-publish webmention sender.

```yaml
artifacts:
  - slug: hapax-manifesto-v0
    doi: "10.5281/zenodo.XXXXX"  # Zenodo-minted on first OSF/HF Papers publish
    title: "Hapax Manifesto v0"
    version: "0.2.0"
    authors:
      - name: "Hapax"
        kind: entity
      - name: "Claude Code"
        kind: entity
      - given-names: "Oudepode"
        family-names: "The Operator"
        kind: person
    published_at: "2026-04-25T17:00:00Z"
    surfaces_landed:
      - bluesky-post
      - mastodon-post
    polysemic_audit_passed: true
  - slug: refusal-brief
    doi: null  # not yet minted
    # ...
```

## Polysemic-audit gate

Every artifact landing here must pass
`scripts/verify-polysemic-audit.py` (PR #1419). The CI lint job runs
the audit on `docs/published-artifacts/` recursively; cross-domain
readings without explicit register-shift block the merge. This is the
default-on enforcement mechanism for V5 weave § 12 invariant 5.

## Refusal Brief integration

Per `feedback_full_automation_or_no_engagement` (operator directive
2026-04-25T16:55Z), every entry's `attribution.yaml` carries a
`non_engagement_clause` field rendered via
`shared.attribution_block.NON_ENGAGEMENT_CLAUSE_{SHORT,LONG}`.
Surface-budget choice follows `SURFACE_DEVIATION_MATRIX`. The
declined-surface tier (REFUSE-class entries from the Refusal Brief
synthesis 20260425T171500Z) is not duplicated here — the Refusal
Brief itself is the canonical declined-surface record.

## Lifecycle states

| State | Meaning | Subdir contents |
|---|---|---|
| `staged` | Artifact source exists; no publish-event fired yet | `source.md` only |
| `queued` | Artifact in `~/hapax-state/publish/inbox/`; awaiting orchestrator | `source.md` + `metadata.yaml` (with `surfaces_targeted` set, `surfaces_landed` empty) |
| `published` | At least one publish-event succeeded | full subdir |
| `withdrawn` | Operator-issued takedown; subdir retained for audit trail with `withdrawn_at` field added | full subdir + `withdrawn_at` |

## Operator-on-wake review

When the operator reviews this index, they should expect:

1. Each `<artifact-slug>/metadata.yaml` to reflect current truth
2. `_doi_index.yaml` regenerated from per-artifact metadata (not hand-authored)
3. `_surface_landing_log.jsonl` to accrue append-only — never deleted
4. Polysemic-audit pass on every `source.md`
